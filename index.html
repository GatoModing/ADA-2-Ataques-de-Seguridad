The Four Attack Types

Click each card to flip and reveal examples

01
🔴

Interruption

The first category, Interruption, directly compromises Availability by destroying or disabling system components to stop legitimate access. This occurs when network services are brought down entirely, halting business operations and causing critical downtime. Practical examples include Distributed Denial of Service (DDoS) attacks that overwhelm servers with massive traffic volumes, physical sabotage such as severing fiber optic infrastructure, or deploying destructive wiper malware designed to wipe master boot records and render operating systems completely unbootable. Availability.

Example

DoS / DDoS Attack

An attacker overwhelms a web server with thousands of fake requests, causing it to crash. Advanced DDoS attacks use botnets — networks of compromised machines flooding targets simultaneously.

Consequences:

  • Complete service downtime
  • Financial losses
  • Reputational damage

02
🟡

Interception

The second category, Interception, focuses on breaching Confidentiality through passive, unauthorized access to secure data streams or files without changing their content. Because the systems continue to operate normally, these attacks often go undetected for long periods. Real-world examples include packet sniffing on unsecured public Wi-Fi networks to harvest unencrypted credentials, wiretapping communication lines to capture active voice data, or quietly cloning a database file containing sensitive client records while leaving the original file intact. Confidentiality.

CIA: Confidentiality

Example

Man-in-the-Middle (MITM)

A hacker sets up a rogue Wi-Fi hotspot in a coffee shop. When users connect, the attacker eavesdrops using packet sniffing or ARP spoofing, capturing passwords and financial data.

Consequences:

  • Theft of sensitive data
  • Corporate espionage
  • Identity theft

03
🟠

Modification

The third category, Modification, actively targets Integrity by intercepting data and altering, tampering with, or destroying its contents before it reaches the intended recipient. This tricks the receiving system or user into trusting corrupted data, leading to severe operational or financial errors. Common examples include Man-in-the-Middle (MitM) financial attacks where an attacker swaps destination bank account numbers during transit, database record alteration to artificially manipulate pricing or user balances, and web defacement where a server is breached to overwrite the site's underlying source code. Integrity.

CIA: Integrity
Click to see examples →
Example

Cross-Site Scripting (XSS)

A hacker injects malicious JavaScript into a legitimate website. When users visit, the script executes in their browsers — stealing session cookies, redirecting to phishing sites, or altering content.

Consequences:

  • Corruption of critical data
  • Unauthorized transactions
  • Spread of misinformation

04
🟢

Fabrication

The fourth category, Fabrication, attacks Authentication by generating entirely counterfeit data, messages, or assets within a system to impersonate a legitimate entity. The system or user is deceived into accepting unauthorized commands or information as genuine from the start. Key examples include email spoofing to falsify mail headers for targeted phishing campaigns, DNS cache poisoning to quietly route unsuspecting users to cloned malicious servers, and log or transaction injection where fake entries are inserted into ledgers to authorize illegitimate payments or cover tracks. Authentication.

CIA: Authentication
Click to see examples →
Example

Phishing + IP Spoofing

A hacker sends emails pretending to be a legitimate bank, creating a fake identical website. Users enter credentials which are stolen. SQL Injection can also insert false records into databases.

Consequences:

  • Fraudulent transactions
  • Identity impersonation
  • Compromised accounts

// PREVENTION MEASURES

How to Defend Against Each Attack

01

Interruption Defense

🛡

Deploy firewalls and Intrusion Prevention Systems (IPS)

🌐

Use Content Delivery Networks (CDN) to distribute traffic

💾

Maintain regular system backups and disaster recovery plans

⚙

Implement rate limiting and traffic filtering

☁

Use cloud-based DDoS mitigation services

02

Interception Defense

🔒

Encrypt all communications using SSL/TLS protocols

📡

Avoid using untrusted public Wi-Fi networks

🔐

Use Virtual Private Networks (VPNs) for secure connections

🔄

Regularly update software and security patches

✅

Implement strong authentication mechanisms

03

Modification Defense

🔍

Implement Intrusion Detection Systems (IDS) and IPS

✍

Use data encryption and digital signatures to verify integrity

🚪

Apply strict access controls and privilege management

🧹

Validate and sanitize all user inputs in web applications

#

Use checksums and hash functions to detect unauthorized changes

04

Fabrication Defense

📝

Use digital signatures to verify sender authenticity

📲

Implement multi-factor authentication (MFA)

🧱

Deploy firewalls and email filtering systems

🎓

Educate users about recognizing phishing attempts

📌

Use certificate pinning and verify SSL/TLS certificates

// PERSONAL CONCLUSION

Final Thoughts

After analyzing these four fundamental attack categories, it becomes clear that network security is not just about preventing unauthorized access — it's about protecting the entire information lifecycle.

Interception teaches us the importance of confidentiality through encryption. Interruption reminds us that availability is crucial for business continuity. Modification highlights why integrity verification is essential, and Fabrication demonstrates the critical need for strong authentication mechanisms.

The most effective defense is a layered security approach that combines technical solutions (encryption, firewalls, IDS) with human awareness (training, phishing recognition). As cyber threats continue to evolve with AI-enhanced attacks, staying informed and proactive is no longer optional — it's a necessity for every individual and organization connected to the internet.
